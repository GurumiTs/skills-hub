# Testing Playbooks

此目錄存放測試策略、Edge Case、測試資料、回歸測試與測試報告格式。

## 原則

* 測試案例必須對應 Acceptance Criteria。
* 未執行的測試不得標示為通過。
* 測試失敗或 blocked 項目必須回交 Developer Agent 或 DB Agent。

## 建議後續文件

| 文件 | 用途 |
|---|---|
| `testing-strategy.md` | 測試策略與測試分層 |
| `edge-cases.md` | Edge Case 設計原則 |
| `test-report.md` | 測試報告格式 |
