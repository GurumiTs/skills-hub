# Skills Hub - Gemini CLI Project Context

## Working Agreement (Change Control)

當任務涉及「實體文件變更」時（包含但不限於：新增/刪除/修改任何檔案、覆寫設定檔、更新 package-lock、重構搬移檔案、批次格式化）：

1) **先不要動手修改任何檔案**（包含自動套用修正、產生檔案、或執行會改動檔案的指令）。
2) 先提供一份「變更提案」，內容必須包含：
   - 風險評估（可能造成的破壞、相容性、資安/權限、回滾難度）
   - 變更清單（每個檔案：新增/修改/刪除、修改重點）
   - 變更摘要（你打算怎麼改、為什麼）
   - 驗證方式（我該怎麼測、用哪些指令或檢查點確認沒壞）

3) 請在提案最後問我一句：「是否要我開始套用變更？」
4) **只有在我明確回覆同意（例如：OK/同意/請開始/Apply）後**，你才可以開始進行實際變更。
5) 套用完後，請回報：
   - 實際變更檔案列表
   - 重要差異摘要（我應該注意的地方）
   - 回滾方式（如果出問題怎麼退回）
